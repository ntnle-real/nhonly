Define `PacingState` type:
```typescript
export interface PacingState {
  currentFragmentIndex: number; // 0–7, which sentence is being revealed now
  visibleFragments: Set<number>; // indices of fragments currently on screen (1–7 visible, 8 triggers fade)
  isTerminalFading: boolean; // true once fragment 8 ("He jumps...") is revealed, triggers text fade
  fadeProgress: number; // 0–1, for terminal fade animation
  userAdvancedAt: number; // timestamp of last user action (ms)
}
```

Define a contract function `computeNextPacing()` that takes:
- current PacingState
- user scroll position (or tap action)
- returns: updated PacingState

Logic:
- Fragment 1 shows immediately on page load
- User must scroll/tap to reveal each subsequent fragment
- Fragments 2–7 accumulate on screen (visible fragments = previous + new)
- Fragment 8 ("He jumps...") becomes visible, then all text begins fading
- Terminal state: fade complete, only ocean particles remain

Document the interaction model: "User scrolls or taps to reveal one sentence at a time. Text accumulates. After final sentence, text fades, leaving only the particle ocean."
