## Key File Details

### `src/routes/+page.svelte` (Main Application)
- **Lines of Code:** ~163 (including markup and styles)
- **Responsibilities:**
  - Recording UI state machine (idle ‚Üí recording ‚Üí stopped)
  - User input handling (buttons, text input)
  - Timer display during recording
  - Title input for archived stories
  - Language toggle
  - Status message display
- **Pattern:** Monolithic component (no child components)
- **Svelte Features Used:**
  - `$state` rune for reactive variables
  - Conditional rendering with `{#if}`
  - Event binding with `on:click` (via `onclick`)
  - Data binding with `bind:value`
  - `onMount` lifecycle hook

### `src/lib/recording.ts` (Audio Capture Service)
- **Lines of Code:** ~91
- **Exports:**
  - `startRecording()` - Initializes MediaRecorder from microphone
  - `stopRecording()` - Stops recording, returns Blob
  - `getElapsedMs()` - Returns recording duration
  - `isRecording()` - Returns current state
  - `RecordingSession` interface (unused)
- **State:** Module-level variables (`mediaRecorder`, `audioChunks`, `startTime`, `elapsedMs`)
- **Error Handling:** Throws on getUserMedia failure, Promise-based for stop

### `src/lib/archive.ts` (IndexedDB Persistence)
- **Lines of Code:** ~128
- **Exports:**
  - `initDatabase()` - Open/create `nhonly_archive` database
  - `saveStory(title, audioBlob)` - Insert story with timestamp
  - `getAllStories()` - Retrieve all stored stories
  - `getStory(id)` - Retrieve single story by ID
  - `ArchivedStory` interface
- **State:** Module-level variable `db` (IDBDatabase reference)
- **Database Schema:**
  - Object store: `stories`
  - Key path: `id` (auto-incremented)
  - Fields: `title` (string), `audioBlob` (Blob), `timestamp` (number)

### `src/lib/i18n.ts` (Internationalization)
- **Lines of Code:** ~69
- **Exports:**
  - `t(key)` - Get translated string
  - `setLanguage(lang)` - Change language
  - `currentLanguage` - Reactive store
  - `Language` type ('en' | 'vi')
- **Supported Languages:**
  - English (en) - Default
  - Vietnamese (vi)
- **Keys Defined:** 8 UI strings (tell_story, saved_as_you_go, stop_recording, story_title, archive_button, story_archived)
- **Store:** Svelte writable store (session-only, not persisted)

### `src/routes/+layout.svelte` (Root Layout)
- **Lines of Code:** ~12
- **Purpose:** Root wrapper component
- **Contains:** Favicon reference in `<svelte:head>`

### `src/app.d.ts` (Type Definitions)
- **Lines of Code:** Variable (auto-generated by SvelteKit)
- **Purpose:** Global ambient type definitions
- **Content:** SvelteKit-generated types for `$lib` alias, `$env`, etc.

### `svelte.config.js` (SvelteKit Configuration)
- Configures adapter (currently auto-detect)
- Loads SvelteKit Vite plugin

### `vite.config.ts` (Build Configuration)
- Configures Vite build process
- Loads SvelteKit plugin for Svelte file handling

### `tsconfig.json` (TypeScript Configuration)
- Compiler options for strict type checking
- Path aliases (e.g., `$lib` ‚Üí `src/lib`)
- Svelte type support

## File Organization Principles

1. **Routes in `src/routes/`** - SvelteKit convention
2. **Shared services in `src/lib/`** - SvelteKit convention
3. **One component per page** - Currently all features in `+page.svelte`
4. **One service per concern** - recording, archive, i18n separated

## Naming Conventions

- **File names:** lowercase, kebab-case for components and modules
- **Route files:** `+page.svelte`, `+layout.svelte` (SvelteKit convention)
- **TypeScript interfaces:** PascalCase (`RecordingSession`, `ArchivedStory`, `Language`)
- **Functions:** camelCase (`startRecording`, `saveStory`, `initDatabase`)
- **Constants:** UPPER_SNAKE_CASE (`DB_NAME`, `STORE_NAME`)
- **Store variables:** camelCase with `$` prefix when used (`$currentLanguage`)
