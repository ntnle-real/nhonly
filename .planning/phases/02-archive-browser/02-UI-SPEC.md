## Spacing Scale

Reused from Phase 1 (Tailwind v4 default scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline spacing |
| sm | 8px | Compact element spacing, list item margins |
| md | 16px | Default element padding, card gaps |
| lg | 24px | Section padding, list-to-footer gaps |
| xl | 32px | Layout gaps, page-level vertical spacing |
| 2xl | 48px | Major section breaks (hero to list) |
| 3xl | 64px | Full-page margins (future multi-page layouts) |

**Exceptions:** Touch targets minimum 52px height (established in Phase 1, maintained for consistency).

**New constraint for Phase 2:** Story list items use md/lg spacing to maintain visual breathing room for elderly users with larger line-spacing preferences.

## Typography

Reused from Phase 1:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | List items, timestamps, playback labels |
| Label | 14px | 600 (semibold) | 1.4 | Button labels, metadata tags |
| Heading | 20px | 600 (semibold) | 1.2 | Section headers, story titles in list |
| Display | 28px | 600 (semibold) | 1.2 | Page headline (archive hero), empty state headline |

**Font families:** Lora (display), Be Vietnam Pro (body/label).

**New convention for Phase 2:** Story titles in list use "Heading" (20px) to create visual hierarchy. Display font (Lora) reserved for reverent moments: empty state, playback success confirmations, deletion confirmations.

## Color

Dominant palette reused from Phase 1:

| Role | Value | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | Teal Deep | #0a1f1f | List background, story containers |
| Secondary (30%) | Teal | #1b6b7b | Subtle hover states, progress bar base |
| Accent (10%) | Teal Light | #2a8fa3 | Primary button highlights, play button, progress indicator |

| Tertiary | Amber | #c8974a | Section headers, date labels (nostalgic warmth) |
| Destructive | Rust | #FF773E | Delete button, destructive confirmation (matches Phase 1) |

**Accent reserved for:**
- Play button (primary interaction)
- Progress scrubber active state
- Active story selection highlight
- "Archive" button on landing (new in Phase 2)

**Semantic colors:**
- Rust (#FF773E): Delete button, delete confirmation heading
- Teal Light (#2a8fa3): Success states (story deleted, playback finished)
- White/90: Disabled states (opacity 0.5)

**New constraint:** Empty state uses Lora display font at 28px to create reverent, inviting tone (not clinical "no results").

## Interaction Contract

### Archive List Scene

**User arrives at archive:**
1. List fades in via opacity (450ms, phase 1 pattern)
2. Each story item is interactive
3. Tapping story title or play button → transitions to playback
4. Tapping delete button → shows confirmation modal (no navigation)
5. Language toggle persists selection to localStorage (phase 1 pattern)

**Expected states:**
- Empty (no stories)
- Populated (1+ stories, sorted newest first)
- Loading (IndexedDB read in progress — optional, if fast skip this)
- Error (IndexedDB read failed)

### Playback Scene

**User taps play on a story:**
1. Playback scene fades in (450ms)
2. Audio begins playing immediately
3. Progress bar/timer updates in real-time
4. User can pause by tapping play/pause button
5. User can replay from start via replay button
6. User can close via X button to return to archive list
7. When audio finishes → automatic return to archive list (2s delay, or immediate with "replay" prompt)

**Expected states:**
- Playing
- Paused
- Finished (auto-return)
- Error (audio corruption or IndexedDB read failure)

### Delete Interaction

**User taps delete button on story item:**
1. Modal overlay appears (fade in 250ms)
2. Focus trapped in modal
3. User must choose "Delete" or "Keep"
4. If "Delete":
   - Modal fades out
   - Story item removed from list
   - Toast appears: "Story deleted" (sand/amber color, fade in/out 3s)
   - Return to archive list after toast
5. If "Keep":
   - Modal fades out
   - Return to archive list
   - Story item remains highlighted (brief blue flash)

## Scene Transitions & Navigation

**Landing Page → Archive Button:**
- New secondary CTA below primary "Begin when you are ready"
- Label: "Archive" (English) / "Lưu trữ" (Vietnamese)
- Style: Secondary button (white/10 bg, white text, 52px height, 24px horizontal padding)
- Tapping → navigates to `/archive` route

**Archive List → Playback:**
- Tapping story or play button → `/archive/{story_id}` or internal scene state
- Fade opacity 0 → 1 (450ms)

**Playback → Archive List:**
- Tapping close button OR audio finishes → `/archive` (fade)
- Auto-return after 2 seconds of silence

**Archive List → Delete Confirm:**
- Modal overlay, no route change
- Confirmation is local state, not navigation

**Archive (empty) → Record:**
- CTA "Record your first story" → `/` (landing → immediate record start)
- Or navigate to record route directly

## Accessibility & Touch Targets

**Phase 2 Inherits from Phase 1:**
- All buttons: minimum 52px height (44px standard + 8px padding)
- All text: white/90 default (high contrast on teal-deep)
- Font sizes: 16px body minimum (readable for elderly users)
- Line heights: 1.5 body (breathing room)
- Touch spacing: 8px (md) minimum between interactive elements

**New constraints for Phase 2:**
- Story list items: each row is full-width touch target (not just title)
- Delete button icon: 18-20px emoji (large enough to tap accurately)
- Play button icon: 20px emoji (centered in button, clear affordance)
- Modal buttons: 50% width, no gap (full-width button layout)

**Keyboard accessibility:**
- Tab order: Language toggle → Story items (in list order) → Archive list actions
- Enter activates buttons
- Escape closes modals and returns to list
